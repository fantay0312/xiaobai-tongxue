# 学伴智舱：面向学生全过程的 AI 实训与成长智能体

## 1. 项目概要

**项目名称**：学伴智舱  
**参赛方向**：学习支持类智能体  
**主题契合**：AI 赋能教·学·管中的“学”与“教辅分析”场景  
**目标用户**：高校学生、任课教师、教学管理人员  
**推荐 Demo 场景**：《Python 数据分析》课程实训与学习成长追踪

本项目面向学生学习全过程，构建一个具备学习诊断、个性化路径规划、实训任务生成、过程式陪练、自动评测和成长档案更新能力的 AI 智能体系统。系统不是传统问答机器人，而是通过多智能体协作、知识库检索、工具调用和学生画像更新，形成“诊断 - 规划 - 实训 - 评测 - 反馈 - 再规划”的学习闭环。

## 2. 核心问题

高校学生在专业课程学习中常见问题包括：

- 不清楚自己真正薄弱的知识点。
- 学习计划空泛，无法转化为可执行任务。
- 实训任务与真实场景脱节。
- AI 工具容易直接替学生完成答案，缺少过程引导。
- 教师难以及时掌握学生个体和班级层面的学习状态。

学伴智舱的核心价值是：让 AI 不只是“给答案”，而是帮助学生形成可验证的学习过程和能力成长证据。

## 3. 作品定位

### 3.1 一句话介绍

学伴智舱是一个面向学生全过程学习的 AI 实训与成长智能体，能够根据学生作业、错题、代码和课程目标，自动诊断能力短板，生成个性化学习路径和真实实训任务，并通过自动评测持续更新学生成长档案。

### 3.2 与普通 AI 答疑工具的区别

| 对比项 | 普通 AI 答疑 | 学伴智舱 |
|---|---|---|
| 服务方式 | 单轮问答 | 全流程学习闭环 |
| 学习依据 | 用户当前问题 | 学生画像 + 课程知识库 + 提交记录 |
| 输出内容 | 答案或解释 | 诊断、路径、任务、反馈、档案 |
| 教育价值 | 偏即时解惑 | 偏能力成长和过程证据 |
| 技术体现 | 文本生成 | 多 Agent 协作 + RAG + 工具调用 + 状态更新 |

## 4. 总体框架

系统采用“多智能体编排 + RAG 知识库 + 工具调用 + 学生画像”的架构。

```text
前端 Demo
  ├─ 学生画像页
  ├─ AI 诊断页
  ├─ 实训任务页
  ├─ 自动评测页
  └─ 教师学情看板

智能体编排层
  ├─ 总控 Agent
  ├─ 学习诊断 Agent
  ├─ 学习路径规划 Agent
  ├─ 实训任务生成 Agent
  ├─ 过程式陪练 Agent
  ├─ 作业评测 Agent
  └─ 成长档案 Agent

能力支撑层
  ├─ 课程知识库 RAG
  ├─ 学生画像库
  ├─ 作业/代码分析工具
  ├─ 代码运行或模拟判题工具
  ├─ 报告生成工具
  └─ 学习风险提醒工具

数据层
  ├─ 课程大纲
  ├─ 实验指导书
  ├─ 评分标准
  ├─ 常见错题
  ├─ 学生提交记录
  └─ 学习成长档案
```

## 5. 使用框架

### 5.1 智能体框架

本项目建议使用以下框架思想实现：

- **多 Agent 工作流**：将诊断、规划、陪练、评测、档案更新拆成独立智能体。
- **RAG 检索增强生成**：让回答、诊断和评测基于课程资料、评分标准和常见错误。
- **工具调用机制**：让智能体调用代码运行、作业分析、画像更新、报告生成等工具。
- **状态化学生画像**：记录学生掌握度、薄弱点、学习行为和阶段变化。
- **教育安全约束**：不直接代写作业，采用提示式、过程式、分层引导。

### 5.2 平台实现建议

可以使用 WorkBuddy、Dify、Coze 或自建 LangChain/LangGraph 原型实现。若以快速参赛 Demo 为目标，推荐优先使用 WorkBuddy 进行智能体工作流搭建，再用轻量前端展示完整流程。

### 5.3 推荐技术组合

| 模块 | 推荐实现 |
|---|---|
| 智能体编排 | WorkBuddy 工作流 / Dify Workflow / LangGraph |
| 大模型能力 | 通用大语言模型 API |
| 知识库 | 平台知识库 / 向量数据库 |
| 前端 Demo | React / Vue / 静态 HTML 原型 |
| 数据存储 | JSON / SQLite / 平台变量 |
| 代码评测 | Python 沙箱 / Mock 判题接口 |
| 报告生成 | LLM 模板化生成 |

## 7. 智能体结构

### 7.1 总控 Agent

负责识别用户意图，并调度不同子智能体。

主要职责：

- 判断用户输入属于诊断、规划、答疑、实训、评测还是报告生成。
- 决定是否需要检索知识库或调用工具。
- 汇总多个智能体结果，输出统一结论。

### 7.2 学习诊断 Agent

负责根据学生提交内容分析学习状态。

主要职责：

- 提取学生已掌握知识点。
- 识别薄弱知识点。
- 分析错误原因。
- 给出知识点掌握度评分。
- 生成后续学习建议。

### 7.3 学习路径规划 Agent

负责将诊断结果转化为可执行学习计划。

主要职责：

- 生成 3 天、7 天或 14 天学习路径。
- 为每一天配置知识点、任务、资料和检查方式。
- 根据学生表现动态调整计划。

### 7.4 实训任务生成 Agent

负责把课程知识点转化为真实任务场景。

主要职责：

- 生成校园、专业或行业相关任务。
- 设定学生角色、任务背景、输入数据和提交要求。
- 配置评分标准和拓展挑战。

### 7.5 过程式陪练 Agent

负责引导学生完成任务，而不是直接给最终答案。

主要职责：

- 判断学生卡住的位置。
- 用问题和小提示引导学生。
- 提供相似例子。
- 控制提示强度，避免直接代写。

### 7.6 作业评测 Agent

负责根据评分标准评估学生提交结果。

主要职责：

- 判断是否满足任务要求。
- 检查代码、报告或答案中的问题。
- 给出评分、扣分依据和修改建议。
- 标记相关知识点掌握状态。

### 7.7 成长档案 Agent

负责把每次学习行为沉淀为长期画像。

主要职责：

- 更新知识点掌握度。
- 记录新增掌握和仍需加强内容。
- 生成学生端成长建议。
- 生成教师端学情提示。

## 9. 工具调用设计

| search_knowledge | 关键词、课程、知识点 | 相关知识片段 | 支撑 RAG 检索 |

## 10. Demo 演示流程

推荐演示主线：

```text
1. 学生进入系统，选择《Python 数据分析》课程。
2. 学生上传一份存在问题的作业代码。
3. 学习诊断 Agent 分析出薄弱点：Pandas groupby、数据可视化、异常处理。
4. 学习路径规划 Agent 生成 7 天补强路径。
5. 实训任务生成 Agent 生成“校园消费数据分析”任务。
6. 学生提交分析代码和结论。
7. 作业评测 Agent 调用评测工具，输出得分、问题和修改建议。
8. 成长档案 Agent 更新学生画像。
9. 教师端看板展示该学生和班级整体学情。
```

## 12. 创新点

1. **学习数字孪生**：为学生构建动态能力画像，而不是只回答单次问题。
2. **全过程学习闭环**：覆盖诊断、规划、实训、陪练、评测、档案更新。
3. **多 Agent 协同**：不同智能体分别承担诊断、规划、评测等专业职责。
4. **真实任务驱动**：将知识点转化为贴近专业和校园场景的实训任务。
5. **过程式陪练机制**：避免直接代写作业，强调提示、追问和自主完成。
6. **可验证成长证据**：每次学习行为都沉淀为画像变化和报告依据。
7. **学生端与教师端联动**：既服务个性化学习，也支撑教师学情分析。

## 13. 技术可行性

本项目可以以原型系统方式提交。比赛阶段无需接入真实教务系统，可使用课程样例数据、模拟学生画像和 Mock 工具接口完成演示。

可行性依据：

- 知识库内容可由课程大纲、实验指导书、评分标准和错题样例构建。
- 多智能体流程可通过 WorkBuddy 等平台编排。
- 学生画像可先使用 JSON 或平台变量存储。
- 代码评测可先使用模拟运行结果，后续再接入真实 Python 沙箱。
- 教师看板可基于样例班级数据展示统计结果。
