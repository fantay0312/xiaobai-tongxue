缺故事感和游戏感——这个判断很可能是对的,费曼模式认知负荷高,光靠"对学习有用"撑不起留存。但动手前先立一条红线,这是之前定过的、现在依然成立:**不做积分/签到/连胜那套通用游戏化贴皮**。要做的是让故事和游戏性从这个产品**独有的资产**里长出来——你有一个有记忆、会成长、会遗忘、会考试的 AI 学生,这本身就是一个现成的故事引擎,大多数产品想要都要不到。

而且有个好消息:下面所有设计都是**表现层**,状态机、评估、记忆架构一行不用改——叙事层只是给已有事件换了一套渲染方式。

## 一、故事感:师徒物语

### 1.1 世界观:书院

给产品一个统一的空间隐喻——你是一位新晋"先生",小白是你收的第一个弟子:

| 系统模块 | 叙事场景 | 界面文案基调 |
|---|---|---|
| 备课模式 | **书房**(灯下备课) | "明日要讲拷贝之术,先生且温书" |
| 讲解舱 | **学堂**(开讲) | 惊堂木一响,小白坐好 |
| 复盘页 | **批注**(灯下阅卷) | 朱笔批注式的报告 |
| 盲区图谱 | **学问舆图**(见下,游戏感核心) | 水墨风格地图 |
| 战术性遗忘 | **小白的来信** | 信笺 UI |

选书院而不是 RPG/科幻的理由:和《学记》"教然后知困"的叙事锚点同源、和白团子形象兼容、和你的东方美学血统一致,而且在教育类评委面前,"书院"比"打怪升级"高级得多。

### 1.2 小白的梦想(角色动机)

故事=角色+欲望+阻碍。现在小白有角色没欲望,补一句人设就够:

> 小白的梦想:"我想有一天,也能像先生一样,把道理讲给别人听。"

这一句话干三件事:给每次"出师"以叙事重量(它离梦想近了一步)、为长线剧情埋钩子(见 1.4)、并且和"教学相长"的主题严丝合缝——**这个故事的主题就是你的产品理论**。

### 1.3 小白的日记(核心叙事装置,强烈建议做)

这是整套设计里我最推荐的一个:把 L2 情景记忆(本来就在存会话摘要)渲染成**小白视角的日记**,每次课后生成一篇,学生可以翻看:

> 七月四日 晴
> 今天先生教我拷贝。先生打了个比方,说 copy() 就像复印文件夹的目录,里面的文件还是同一份——我一下子就懂了!
> 不过我后来想,那 copy 完之后,里面的小列表应该也是新的吧。嗯,应该是这样。
> 先生明天还来吗?

注意第二段:如果学生当天**被误区带偏了**(M1 认同了错误说法),日记里小白就会自信地写下那个**错误理解**。学生翻日记时猛然发现——"等等,这是错的……而且是我教的。"

这是这个产品才做得出来的叙事装置:**盲区揭示不再是报告里的一行红字,而是你亲手写进别人日记里的一个错误**。责任感、愧疚感、纠正欲,全部由故事自然触发。工程上它就是把已有的会话摘要+事件流换一个 prompt 模板渲染,成本半天。

### 1.4 长线剧情弧:结业典礼

当小白在某门课程下出师满 N 个知识点,触发终章剧情——**小白第一次给"小小白"讲课**(过场动画/对话回放,不是给你讲,不违反认知天花板):

> 小白对小小白说:"你看,copy() 就像复印文件夹的目录……"
> ——它用的是**你教它的那个比喻**。

你的金句在它嘴里传承下去。"弟子成了先生",教学相长的主题在此闭环,这是整个产品的情感 payoff,也是答辩视频的完美结尾镜头。

## 二、游戏感:五个机制(全部锚定真实评估)

### 2.1 观战 Boss 战:送小白去考试

把已有的"小白测验"从一个后台分数变成**观战玩法**:你教完之后,亲手把小白送进考场,考试期间**你不能帮它**,只能看着它逐题作答、边答边嘀咕——

> 第 2 题……"这个先生讲过!是引用共享!" ✅
> 第 4 题……"呃,这个……先生好像没细说……"(犹豫 8 秒)❌

这是养成游戏里"看自己的宝可梦上场"的经典张力,但**考试本身是真实评估**,一点信用都没损失。它答错时那句"先生好像没讲过",比任何盲区报告都扎心且精准。UI 成本:一个逐题播放的动效 + 小白内心 OS 的 prompt 模板。

### 2.2 心魔系统:误区的叙事化

误区库整体包装成"心魔"——学问路上的迷障。注意时机纪律:**注入当下绝不提示**(否则评估隐身失效),戏剧化留到复盘的"战报"里揭示:

> ⚔️ 本课遭遇心魔 ×2
> 「浅拷贝之惑」——被你一招击退(内存图反驳,漂亮)
> 「嵌套之迷」——小白被它拐走了……(已标记,待你去救)

"救回被心魔拐走的小白"=补学微路径的叙事外壳。同一个流程,动机完全不同了。

### 2.3 迷雾舆图:盲区图谱的游戏化重构

盲区图谱直接套用游戏里最经典的**战争迷雾**机制,语义完美对位:

| 图谱状态 | 舆图表现 |
|---|---|
| 未接触的知识点 | 迷雾笼罩 |
| 讲过但有盲区 | 雾散一半,心魔盘踞 |
| 出师 | 云开雾散,亮起灯火 |
| 掌握度衰减 | **雾气缓缓重新聚拢**(衰减可视化,自然引出复习) |

"消灭迷雾"的收集欲是刻在玩家 DNA 里的,而这里每一格迷雾的消散都对应真实的掌握度事件。

### 2.4 双图鉴收集

- **锦囊图鉴**:金句类比做成卡牌(卡面=小白+那句比喻),只有 checklist 命中且质量达标才生成——不可刷;
- **心魔封印录**:每纠正一个误区,该心魔卡片被"封印"入册,集齐一门课的所有心魔=课程全清成就。

图鉴的完成度天然就是课程掌握度的另一种投影,收集系统和评估系统是同一份数据。

### 2.5 称号系统(取代等级和积分)

称号全部由真实指标派生,描述的是**你的教学风格**而非肝度:

| 称号 | 触发条件(真实指标) |
|---|---|
| 初执教鞭 | 第一次开讲 |
| 循循善诱 | 卡壳救援后继续讲完率 ≥80% |
| 一点就通 | 平均出师轮数 ≤3 |
| 降魔真人 | 心魔一次击退率 ≥90% |
| 桃李满门 | 结业典礼达成 |

## 三、仪式感清单(便宜但高感知的 juice)

开讲时惊堂木音效一响、小白起立行礼;出师时颁发**结业证书,落款是学生的名字——"授业师:XXX"**;舆图点亮用水墨晕开动画;小白的来信用信笺展开动效。每个都是几小时的活,感知度极高。

## 四、映射表:叙事层 ↔ 已有架构(给团队和评委看)

| 叙事元素 | 底层组件 | 新增工程量 |
|---|---|---|
| 小白的日记 | L2 情景记忆 + 事件流 | 一个渲染 prompt |
| 观战 Boss 战 | 小白测验 | 逐题播放 UI + OS 模板 |
| 心魔战报 | 误区注入/纠正事件 | 复盘页文案层 |
| 迷雾舆图 | 盲区图谱 + 掌握度衰减 | 前端皮肤 |
| 锦囊/封印图鉴 | 金句收藏 + mc_corrected 事件 | 卡片组件 |
| 小白的来信 | 战术性遗忘调度 | 信笺 UI |
| 结业典礼 | topic_mastered 计数 | 一段过场模板 |

这张表就是答辩防御:"我们的叙事层是纯表现层,每个故事元素背后都是一个真实评估事件——**故事不是糖衣,是数据的另一种可视化**。"

## 五、不做什么(红线重申)

每日签到、连胜 streak、体力值、抽卡随机奖励、排行榜——全部不做。它们要么制造焦虑、要么可刷、要么引入和学习证据无关的随机性,任何一条都会腐蚀"所有反馈不可伪造"这个立身之本。这个产品的游戏感必须是**巫师三式的(叙事驱动)而不是老虎机式的(多巴胺驱动)**。

## 六、落地优先级(基于你们已做了一部分)

| 优先级 | 内容 | 理由 |
|---|---|---|
| P0 | 场景命名+文案全面书院化、心魔战报、小白的日记 | 几乎纯文案层,一两天,气质立变 |
| P0 | 出师证书+惊堂木仪式 | 便宜,Demo 高光帧 |
| P1 | 观战 Boss 战、迷雾舆图 | 各需一个专门 UI,是游戏感的两根柱子 |
| P2 | 双图鉴、称号 | 锦上添花 |
| P2 | 结业典礼长线剧情 | Demo 里可以只放一张概念图讲愿景 |
# 「小白同学」叙事层落地包 v1.0

> 定位：本文档是《智能体项目完整方案》叙事/游戏化章节的落地资产包——可直接进开发排期的文案、Prompt 模板与交互规格。红线不变：不做签到/积分/连胜/抽卡/排行榜，所有叙事均为真实评估事件的渲染层，状态机、评估、记忆架构一行不改。

## 0. 一条贯穿所有设计的验收规则

每个叙事元素都必须能回答一个问题："我渲染的是哪个底层事件？"回答不出来的设计一律砍掉。本文档每节末尾都标注了数据来源，答辩时这就是防御工事。

## 1. 小白的日记（P0）

### 1.1 三篇标准样例（覆盖三种核心状态）

**样例 A · 顺利的一课**（事件：topic_taught + mc_corrected + golden_analogy 命中）

> 七月四日　晴
>
> 今天先生教我浅拷贝。我本来以为 copy() 会把里面的东西全都复印一遍，先生画了一张内存图给我看——原来 copy() 只是复印了文件夹的目录，里面的文件还是同一份！我把这张图记在心里了。
>
> 先生说明天讲深拷贝。好想快点到明天。

**样例 B · 被心魔拐走的一课**（事件：mc_injected + mc_accepted，未纠正）

> 七月六日　雾
>
> 今天先生讲嵌套列表的拷贝。我问先生，copy() 之后里面的小列表是不是也是新的，先生说是的。
>
> 那我就放心了！以后遇到嵌套列表，直接 copy() 就好，改里面的东西也不会互相影响。我又学会一招，真开心。

关键纪律：全文不得出现任何"这可能是错的"暗示（无红字、无问号语气、无异常样式）。错误必须以小白最自信的语气写下——学生翻到这页时的心理落差就是产品要的那一下。

**样例 C · 遗忘前夜**（事件：mastery_decay 达到阈值，作为"小白的来信"的前奏）

> 七月十八日　阴
>
> 好些天没上深拷贝的课了。今天自己温书，翻到之前的笔记，有个地方忽然想不起来了……deepcopy 遇到自己引用自己的列表，会不会复印个没完？
>
> 先生讲过吗？好像讲过，又好像没有。明天写信问问先生吧。

### 1.2 日记渲染 Prompt 模板

```
你是小白，一个天真好学的 AI 学生。根据今天的课堂记录写一篇日记。

【输入】
- 今日课程摘要（L2 情景记忆）
- 事件流：mc_injected / mc_corrected / mc_accepted / stuck / stuck_rescued / topic_mastered / mastery_decay
- 先生今天的金句或比喻（如有）
- 天气（由系统按 1.3 规则传入，不要自己决定）

【规则】
1. 第一人称小白视角，100~200 字，语气天真、认真、对先生有孺慕之情
2. 若事件流含 mc_accepted（误区被认同且未纠正）：必须把该错误理解自信地写进日记，
   作为"今天的收获"，禁止任何暗示它可能有误的措辞
3. 若含 mc_corrected：写成"我本来以为……先生给我讲明白了"的顿悟结构
4. 若含 stuck_rescued：写小白当时的困窘和先生换讲法后的那一下"懂了"
5. 至少引用一次先生当天的原话或比喻（从金句输入中取，不得改写核心意象）
6. 结尾一句对明天/下次课的期待或小小的疑问
7. 只能使用先生教过的知识和课堂上出现过的信息（认知天花板同样约束日记）
```

### 1.3 天气 = 状态投影（新增微设计）

日记的天气不是随机装饰，由当日事件流映射，是一个不动声色的掌握度仪表：

| 天气 | 触发条件 |
|---|---|
| 晴 | 顺利出师，或心魔一次击退 |
| 阴 | 卡壳 ≥2 次，或衰减提醒触发 |
| 雾 | 存在未纠正误区（mc_accepted） |
| 雨 | 测验大面积失利（正确率低于阈值） |

学生连翻几页发现"最近总是雾天"，不需要任何红字就感知到了盲区在堆积。

**数据来源**：L2 会话摘要 + 事件流。**新增工程量**：1 个渲染 prompt + 信笺/日记前端组件。

## 3. 心魔系统（P0 文案层 → P1 卡片）

### 3.1 心魔命名法（误区自动起名）

误区入库时由模型自动生成 4~6 字心魔名，后缀按误区类型选用：

| 误区类型 | 后缀 | 例 |
|---|---|---|
| 概念混淆 | 之惑 | 浅拷贝之惑 |
| 想当然过度泛化 | 之执 | 万物皆引用之执 |
| 边界/特例盲区 | 之迷 | 嵌套之迷 |
| 记而不解、背而不明 | 之障 | 死记之障 |

命名 prompt 要求：名字必须点出误区的核心意象，看到名字能回忆起是哪个坑；禁止生成生僻字。

### 3.2 心魔战报模板（复盘页，三种结局）

```
⚔️ 本课遭遇心魔 ×2

「浅拷贝之惑」—— 一招击退。
你用内存图正面反驳，小白当场醒悟。此魔已入封印录。

「嵌套之迷」—— 小白被它拐走了。
它认同了"copy 之后内层也是新的"，此刻正带着这个念头写今天的日记。
［去救小白 →］
```

中间态（缠斗后击退）的文案骨架：

```
「XX 之惑」—— 缠斗三回合后击退。
你第一次的解释没能驱散它，换了比喻之后才成功——
这个心魔对你的第一套讲法免疫，已为你记下。
```

"去救小白"按钮直接进入补学微路径——同一个流程，动机完全不同。

时机纪律重申：**注入当下界面零提示**，一切戏剧化只出现在课后战报，否则评估隐身失效。

**数据来源**：mc_injected / mc_corrected / mc_accepted 事件。**新增工程量**：命名 prompt + 复盘页文案层。

## 4. 观战 Boss 战：送小白赴考（P1）

### 4.1 流程

1. **送考仪式**：确认弹窗——小白背上小书箱，行礼："先生，我去了！"下方一行小字："考场之内，先生不得代答。"
2. **逐题播放**（核心观战体验，见 4.2）
3. **放榜**（见 4.4）

### 4.2 逐题播放规格

每题四拍：

1. 亮题（约 1s）
2. 小白内心独白流式吐出（打字机效果，犹豫处插入"……"停顿）
3. 落笔作答
4. 判定：答对盖一枚朱印，答错落一点墨渍

犹豫时长映射（这是观战张力的来源，必须来自真实数据）：停顿秒数由该知识点掌握度反比决定——满掌握约 1s 干脆作答；半掌握 3~5s；盲区 6~8s 且独白中出现"先生好像没讲……"。

### 4.3 考场独白 Prompt 模板

```
你是考场上的小白，对每道题输出内心独白（20~40 字）+ 最终作答。

【规则】
1. 该知识点先生讲过且你已掌握：干脆利落，引用先生的讲法。
   例："这个先生讲过！copy 只复印目录——是引用共享！"
2. 先生没讲过或讲得含糊：如实犹豫。
   例："呃……这个……先生好像没细说……我猜猜看……"
3. 你带着未纠正的误区：自信地用错误理解作答（并因此答错），
   独白中不得流露怀疑。
4. 认知天花板：禁止使用先生没教过的知识正确作答。
5. 独白仅是过程呈现，判分完全由测验系统决定，独白不得与判分矛盾。
```

### 4.4 放榜

成绩单书院化：朱笔总分 + 逐题批注。所有独白中出现"先生没讲过"的题目，一键落入舆图，标为迷雾/心魔格——盲区图谱的入口从"看报告"变成"收拾考场留下的烂摊子"。

**数据来源**：小白测验（真实评估），独白是纯表现层。**新增工程量**：逐题播放 UI + 独白 prompt。

## 5. 迷雾舆图（P1）

### 5.1 状态与交互

| 图谱状态 | 舆图表现 | 点击行为 |
|---|---|---|
| 未接触 | 迷雾笼罩 | 不可点，hover 提示"此地未开蒙" |
| 讲过但有盲区 | 雾散一半，心魔盘踞 | 弹出心魔卡 →［去救小白］ |
| 出师 | 云开雾散，亮起灯火 | 回放该知识点高光时刻（金句、击退记录、当日日记） |
| 掌握度衰减 | 雾气缓缓重新聚拢 | 进入复习（衔接"小白的来信"） |

### 5.2 关键动效

雾气回拢不做实时动画，做"对比渲染"：每次打开舆图时，从上次快照的雾量过渡到当前雾量，几秒的水墨晕开/聚拢。学生亲眼看着自己上周点亮的灯火被雾重新吞掉一角——复习动机不需要任何推送文案。

**数据来源**：盲区图谱 + mastery_decay 事件。**新增工程量**：前端皮肤 + 快照对比渲染。

## 8. 小白的来信（P0/P1，战术性遗忘的渲染层）

### 8.1 信件模板

```
先生钧鉴：

前日所学「深拷贝」，弟子近来自己温书，忽觉有一处模糊了……
deepcopy 遇到自己引用自己的列表，会不会复印个没完？
弟子越想越乱。

先生得空时，能再与弟子讲讲么？

　　　　　　　　　　弟子　小白　顿首
```

### 8.2 生成规则

信中必须包含两部分：小白**还记得的部分**（复述正确残留，证明它不是全忘了）+ **精确模糊掉的那一点**（来自遗忘调度选中的知识点）。这样复习入口带着一个具体问题进来，而不是一句干瘪的"该复习了"。

**数据来源**：战术性遗忘调度。**新增工程量**：信件 prompt + 信笺 UI。

## 9. 结业典礼脚本（P2，Demo 阶段放概念图即可）

场景：学堂。小白第一次坐在先生的位置上，台下坐着小小白。

对白回放：小白讲课的台词从**你的金句库**中抽取——它用的是你教它的那个比喻，一字未改。过场形式为对话回放/动画，不与你交互，不违反认知天花板。

结尾字幕：

> 教然后知困，学然后知不足。——《学记》

镜头收在证书墙上，落款处是学生的名字。这是答辩视频的最后一个镜头。
